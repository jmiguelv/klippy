Route Consolidation: Home & Chats Index

Goal: Remove the jarring transition between the identity check and the chat workspace.

A

Unified Root (/)

Merge everything into the root route. If no name is found, show the Identity Form. Once named, show the Chat Hero. No redirects needed.

URL: /
[ Identity Form ] → [ Chat Hero ]
B

Identity as Global Gate

Keep /chats as the home. If userName is missing, any route displays the Identity Form as a full-screen overlay. Supports deep-linking.

URL: /chats/123 (Deep Link)
[ Identity Overlay ]
C

Redirect & Render

Keep / as a lightweight landing/redirect. /chats renders either the IdentityForm or ChatHero component based on local state.

/ (Redirect) → /chats
Identity
Hero

Recommendation: Approach A (Unified Root)

This is the cleanest from a user perspective. There's only one "home" URL. We can keep the sidebar visible but "de-emphasized" or empty during the identity phase to maintain structural continuity.

Pros of Approach A

  • Zero redirects = faster perceived load.
  • Consistent layout from second zero.
  • Simplifies +layout.svelte logic.

Cons

  • Requires moving /chats content to root.
  • Needs careful handling of the sidebar transition.
