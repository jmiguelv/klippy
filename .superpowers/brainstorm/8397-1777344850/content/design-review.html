2. Component Fragmentations

A

Duplicated Composer Logic

The query input, settings (Top K, Threshold), and filter handling are duplicated across +page.svelte (chats index) and [id]/+page.svelte (chat instance).

B

Autocomplete Variations

Three different implementations of autocomplete/dropdown logic exist (Home, Chats Index, Chat Instance), leading to varying behavior and UI styles.

C

Filter Chip Inconsistency

Chat instance shows active filter chips; Chats Index does not, even though it parses them into the query string. User bubbles show filters differently than the composer.

3. Key Proposed Improvements

Refactorings

  • Componentize Composer: Create a shared Composer.svelte to unify input logic, filters, and settings.
  • Unify Autocomplete: Extract autocomplete into a reusable action or component with consistent styling.
  • Consolidate Routes: Merge Home and Chat Index into a single workspace-style interface to reduce navigation hops.

UI/UX Fixes

  • Z-Index Audit: Consolidate z-index values into CSS variables to prevent overlap bugs (e.g., autocomplete vs navbar).
  • FOUC Prevention: Move theme detection to a blocking script in app.html to avoid white flashes in dark mode.
  • Mobile Polish: Add backdrop and proper close buttons to the mobile sidebar.
